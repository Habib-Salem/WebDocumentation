Introduction

The purpose of this web documentation report is to effectively provide users a justification and rationale behind the various design descisions taken in the prototype website for the Australian Red Cross Lifeblood. Correspondingly, the various iterations and changes of the website prototype will be documented; aided by the explanation of fundamental design descisions that had led the designers towards making those particular changes.

Reflection on Website Design Proposal

The intended website prototype will effectively use the final mockups (ideated within the website design proposal) as a reference for which it is created upon. Correspondingly, feedback gained previously from the website design proposal itself (through tutors and users) will be also applied as an iteration through the intended website prototype. However, it is notable that the process of feedback and improvement is an ongoing iterative process in which the designers intend to constantly collect and recieve user feedback in order to be analysed and applied to the final intended web prototype for the Australian Red Cross Lifeblood.

Feedback Recieved on Mockups

After analysing feedback from tutors regarding the previous website design proposal, the designer had asked various users to give their feedback on the mockups and potentially suggest any improvements that would ultimately improve the overall user experience and user interface of the intended website prototype. Through this, three participants had volunteered to provide informal feedback of which was to later to be used to improve the various iterations of the website prototype for the Australian Red Cross Lifeblood. Hence, they include:

"I love the colour and aesthetic of the mockups, particularly the landing page, it is really nice!. However if I had to suggest an improvement, I want the navigation bar to have less options, it's too overclustered right now" Jessica Tran, 22
"If there was only a way for you to compress the pages. I feel like there are too many pages when multiple web pages could just be on a singular one" Nathan Lowe, 18
"If you can get the whole 'streamlined' process of donating blood with the progress bar at the bottom, that would be absolutely amazing" Susana Tolentino, 34

Documentation

The following web documentation report will be structured according to the following list:

  1. Landing Page
  2. Blood donation and eligibility details page
  3. Eligible call-to-action page
  4. Ineligible call-to-action page
  5. Centres near me page

Therefore, it is through this scaffold that each prototyped web page will have its unique section in this web documentation report. Correspondingly, it is through each section (that each web page prototype represents) that the designers iteration, rationale and justifcation descisions will be explored; particularly through annotations, screenshots and comparisons between the mockups, prototypes and numerous iterations.
